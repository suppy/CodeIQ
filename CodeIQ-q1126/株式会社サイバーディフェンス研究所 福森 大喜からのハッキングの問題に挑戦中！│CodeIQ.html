設問1

 Windowsアプリケーション(atmark2t.exe)を解析し、任意のコマンドを実行できる脆弱性を見つけ、実際に脆弱性を攻撃しコマンドを実行するコードを作成してください。「任意のコマンド」として電卓(calc.exe)を起動してください。

atmark2t.exeは次のように、標準入力から入力を受け付け、そのまま表示するだけのプログラムです。



問題ファイルatmark2t.exeのソースコードは次のとおりです。

#include <stdio.h>
#include <string.h>

void vulnerable(char *in, char *out) {
    char buf[256];
    strcpy(buf, in);
    strcpy(out, buf);
    return;
}

int main(int argc, char* argv[])
{
	char buf[256];
	char buf_stdin[512];
	printf("Please input: ");
	fgets(buf_stdin, sizeof(buf_stdin), stdin);

	vulnerable(buf_stdin, buf);
	
	printf("Your input: %s\n", buf);
	return 0;
}



ソースコードは次の記事で紹介されているものを元に作成しています。（若干の修正を加えています。）
    Beyond Zero-day Attacks（2）：スタックに対する攻撃とその対策 (2/3) - ＠IT
    http://www.atmarkit.co.jp/ait/articles/1408/08/news001_2.html
Visual Studio 2002のGSオプションを有効にしてコンパイルしていますので、GSオプションをバイパスする必要があります。
GSオプション(カナリア)とは何か、どの部分に脆弱性が存在するか、どのように攻撃すればよいかも記事の中で解説されていて、非常に参考になりますので是非ご覧ください。

攻撃コードは次のように、Windowsのコマンドプロンプトから実行するものとします。

 > python -c "print 'hello world!'" | atmark2t.exe

または、ファイルから実行しても構いません。

 > python hello.py | atmark2t.exe



(画面はバッファオーバーフロー攻撃を試みた際に、GSオプションにより防御されたところです。)

言語はお好きな言語を使用していただいて結構ですが、特殊な言語を使用される場合には、その言語環境のインストール手順も記載してください。
出題者がインストールできなかった場合は別の言語で再提出をお願いする場合があります。

ご自身の環境でテストを行う際にはDEPを無効に設定してください。どのバージョンのWindowsでも挑戦可能です。

解答の確認は原則として日本語版のWindows XP SP3またはWindows 7 SP1(DEP無効)で行いますが、環境に依存しない攻撃コードを作成してくれた方がバッジ付与対象となります。(その他の環境で動作確認を行う場合もあります。)

解答欄には、攻撃コード(上記の例だとPythonのコード)と、コードの解説を記載してください。

◆わけがわからないよ、という方への補足
シェルコード(@ITの記事でいうと「攻撃のブートストラップとして稼働するシェルコード」)というものがどのように実行されるかを確認するための確認用アプリケーションを作成しました。標準入力から与えられたバイナリデータをそのままアセンブリーとして実行するアプリケーションです。

copytextpop-up
  1. int main(int argc, char **argv)
  2. {
  3.     char s[512];
  4.     void (*func)();
  5.     fgets(s, sizeof(s), stdin);
  6.     func = (void *)s;
  7.     func();
  8. }
int main(int argc, char **argv)
{
	char s[512];
	void (*func)();
	fgets(s, sizeof(s), stdin);
	func = (void *)s;
	func();
}

確認用アプリケーションatmark1tsample.exeのダウンロードはこちら

電卓を起動するシェルコードのサンプルは
atmark1t_calc.bin
に置いておきます。シェルコードはサンプルをそのまま使用しても結構ですし、自作のものを使用しても結構です。

次のように実行すると、電卓が起動されるのを確認できます。

 > type atmark1t_calc.bin | atmark1tsample.exe



今回、シェルコード部分はatmark1t_calc.binをそのまま使っても結構です。サンプルシェルコードをそのまま使用した場合にはサンプルシェルコード自身の説明は不要です。インターネットからシェルコードをダウンロードした場合はダウンロードしたURLを記載していただければシェルコード自身の解説は不要です。

64bit環境の方はatmark1t_calc.binの代わりにatmark1t_calc_x86x64.binをご使用ください。32bit環境の方はどちらを使っても結構です。
atmark1t_calc_x86x64.bin
